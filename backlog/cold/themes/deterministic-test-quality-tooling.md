### Theme: Deterministic Test-Quality Tooling (mutation testing + job-payload contract)

_Focus: fill the remaining deterministic-gate rungs so seam/wiring bugs fail the build, not slip through green line-coverage. Sibling to [`production-observability-perf-metrics-tracing.md`](production-observability-perf-metrics-tracing.md) — both make the unsafe/invisible thing deterministic._

**Surfaced 2026-06-11 (user)** after the iii-b-2 thin-payload referenced-attachment regression (#1184): `jobChainOrchestrator` **had** a referenced-attachment test, and line coverage was green — but it covered only the _fat_ payload shape, so a new wire-shape shipped broken. Three green units, one broken cross-service seam. User's framing: unit tests are repeatedly insufficient for seam/wiring bugs; we want **deterministic checks that fail the build**. We already have strong gates (cpd ratchet, test-audit, depcruise, conformance harness, codecov) — this fills the remaining rungs.

**Candidates to evaluate (with honest scope of what each catches):**

1. **Mutation testing — StrykerJS** _(highest-leverage general tool)._ Line coverage measures code-_ran_, not bug-_caught_. Stryker mutates code (flip conditionals, delete statements, swap `??`/`&&`) and checks whether a test fails → grades test _effectiveness_. **Caveat**: catches weak tests, NOT missing code paths — it would not have caught #1184 directly, but it's the best deterministic answer to "are our tests a real net."
   - **Pilot ✅ SHIPPED (#1459, 2026-07-02)** — report-only on `config-resolver` (Stryker 9.6.1 + vitest runner; `pnpm test:mutation` from the package). **First run: 60.71% score, 3m29s** at concurrency 2 under LOW_RESOURCE_MODE (CI-viable). 593 mutants: 360 killed / 218 survived / 15 uncovered. ~⅔ of survivors are pino log-payload noise (91 ObjectLiteral + 55 StringLiteral); **~57 are genuine logic gaps** (22 flipped conditionals, 19 optional-chaining removals, 12 emptied blocks) incl. cascade-tier guards in `ConfigCascadeResolver` (:193 undefined-check, :219 null-check operator). Config gotcha encoded: pnpm's strict layout needs the vitest-runner plugin named explicitly. Premise validated — green line coverage hid a third of behavior changes.
   - **Gap-closing ✅ SHIPPED (#1461, 2026-07-02)** — 23 tests, measured 60.71% → 77.74% raw; logic-class survivors 73 → 4 (all documented equivalents). The split that matters for the ratchet: **logic classes 97.2%** (344/354 killed), noise classes (ObjectLiteral/StringLiteral) 48.9% — and ~half the noise class IS load-bearing (117 killed), so wholesale mutator exclusion loses signal.
   - **Ratchet ✅ SHIPPED (#1463, 2026-07-03) — pilot-package infrastructure complete.** Logger-call ignorer plugin (kept meaningful string/object mutants measured; 76 ignored, survivors 119→58 with zero new tests) + `pnpm ops mutation:check`/`mutation:update-baseline` with the full audit-class ceremony (WHY.md, canary, JSONL summary, configHash drift contract). Baseline-and-hold at the tuned measured **87.81** (grace 1 → floor 86.81); hollow reports (zero measurable mutants) and missing reports both FAIL rather than silently pass. CI: parallel `mutation-tests` job (~3min). **Expanding to another package**: copy config-resolver's stryker.config.mjs + ignorer + `test:mutation` script, add to `MUTATED_PACKAGES` (drift forces the baseline refresh), add its Stryker step to the CI job.
