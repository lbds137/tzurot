### Theme: Deterministic Test-Quality Tooling (mutation testing + job-payload contract)

_Focus: fill the remaining deterministic-gate rungs so seam/wiring bugs fail the build, not slip through green line-coverage. Sibling to [`production-observability-perf-metrics-tracing.md`](production-observability-perf-metrics-tracing.md) — both make the unsafe/invisible thing deterministic._

**Surfaced 2026-06-11 (user)** after the iii-b-2 thin-payload referenced-attachment regression (#1184): `jobChainOrchestrator` **had** a referenced-attachment test, and line coverage was green — but it covered only the _fat_ payload shape, so a new wire-shape shipped broken. Three green units, one broken cross-service seam. User's framing: unit tests are repeatedly insufficient for seam/wiring bugs; we want **deterministic checks that fail the build**. We already have strong gates (cpd ratchet, test-audit, depcruise, conformance harness, codecov) — this fills the remaining rungs.

**Candidates to evaluate (with honest scope of what each catches):**

1. **Mutation testing — StrykerJS** _(highest-leverage general tool)._ Line coverage measures code-_ran_, not bug-_caught_. Stryker mutates code (flip conditionals, delete statements, swap `??`/`&&`) and checks whether a test fails → grades test _effectiveness_. **Caveat**: catches weak tests, NOT missing code paths — it would not have caught #1184 directly, but it's the best deterministic answer to "are our tests a real net."
   - **Pilot ✅ SHIPPED (#1459, 2026-07-02)** — report-only on `config-resolver` (Stryker 9.6.1 + vitest runner; `pnpm test:mutation` from the package). **First run: 60.71% score, 3m29s** at concurrency 2 under LOW_RESOURCE_MODE (CI-viable). 593 mutants: 360 killed / 218 survived / 15 uncovered. ~⅔ of survivors are pino log-payload noise (91 ObjectLiteral + 55 StringLiteral); **~57 are genuine logic gaps** (22 flipped conditionals, 19 optional-chaining removals, 12 emptied blocks) incl. cascade-tier guards in `ConfigCascadeResolver` (:193 undefined-check, :219 null-check operator). Config gotcha encoded: pnpm's strict layout needs the vitest-runner plugin named explicitly. Premise validated — green line coverage hid a third of behavior changes.
   - **Gap-closing ✅ SHIPPED (#1461, 2026-07-02)** — 23 tests, measured 60.71% → 77.74% raw; logic-class survivors 73 → 4 (all documented equivalents). The split that matters for the ratchet: **logic classes 97.2%** (344/354 killed), noise classes (ObjectLiteral/StringLiteral) 48.9% — and ~half the noise class IS load-bearing (117 killed), so wholesale mutator exclusion loses signal.
   - **Ratchet ✅ SHIPPED (#1463, 2026-07-03) — pilot-package infrastructure complete.** Logger-call ignorer plugin (kept meaningful string/object mutants measured; 76 ignored, survivors 119→58 with zero new tests) + `pnpm ops mutation:check`/`mutation:update-baseline` with the full audit-class ceremony (WHY.md, canary, JSONL summary, configHash drift contract). Baseline-and-hold at the tuned measured **87.81** (grace 1 → floor 86.81); hollow reports (zero measurable mutants) and missing reports both FAIL rather than silently pass. CI: parallel `mutation-tests` job (~3min). **Expanding to another package**: copy config-resolver's stryker.config.mjs + ignorer + `test:mutation` script, add to `MUTATED_PACKAGES` (drift forces the baseline refresh), add its Stryker step to the CI job.
   - **Suite-wide expansion — IN PROGRESS (user call 2026-07-03: one pilot package ≠ done).** Roll out per-package using the recipe above, each rollout being: run report-only → gap-close the real logic survivors → tune ignorer if a new noise class dominates → baseline. **cache-invalidation ✅ (2026-07-06)**: 55.51% raw → 88.98% baselined. The rollout hit the anticipated new-noise-class case: 62% of survivors were `logOptions` observability-callback payloads (getLogContext/getEventDescription bodies + their consumption intermediates — built one statement before the logger call, invisible to the logger-call rule) → new `observability-options` ignorer rule (that package's copy only; EXPECTED_IGNORERS is now the cross-package union). The REAL gap class was the subscribe-path wire contract: no service test delivered raw pub/sub messages, so validator specs and dispatch conditionals were unasserted → uniform `subscribe dispatch (wire-contract validation)` blocks across all 8 per-domain services + strict key-count cases for the hand-rolled validators. Remaining packages (conversation-history, identity, clients); the three services still need a viability measurement first (suite size × mutant count may blow the ~3min CI budget — measure before committing, consider per-service jobs or a changed-files scope). Note: config-resolver rebaselined 87.81→87.17 at the same update (new #1506 resolver-reader mutants; within the old grace) — its new survivors are a gap-close candidate for that package's next touch.
2. **Job-payload contract / property suite at the bot→gateway→worker BullMQ seam** — ✅ **SHIPPED 2026-07-06** _(targeted at the #1184 class)._ Three layers: (a) **property tier** (fast-check 4.8, adopted after live research; hand-rolled arbitraries in `test-utils/jobContextArbitraries.ts` — the Zod-4 bridge ecosystem is dead/churny; pinned seed + `FC_SEED` override for deterministic CI) — `jobChainOrchestrator.property.test.ts` asserts no-drop/no-dupe of reference-stamped preprocessing children and schema+envelope round-trip for EVERY generated context; (b) **fixture tier** — envelope-minimal + envelope-referenced-attachments fixtures added to the bullmq-job-chain contract pair (the old fixtures were legacy-only: shapes bot-client never ships); (c) **pipeline tier** (`BullMQJobChainPipeline.consumer.contract.test.ts`) — runs the committed producer fixtures through the REAL ValidationStep→NormalizationStep→ContextStep (PGLite assembler) + DependencyStep re-association keying, and pins the three-tier narrowing (legacy fixtures: schema-tolerated but ContextStep-REJECTED). **Verified the suite catches the resurrected #1184 bug** (fallback hand-broken → property AND fixture layers both failed, with a shrunk minimal counterexample).
3. **Evaluate Pact (consumer-driven contracts)** — ❌ **RULED OUT 2026-07-06** (live research during candidate 2's adoption pass): Pact is consumer-driven contract testing for independently-deployed HTTP pairs with a broker; wrong tool for an in-repo BullMQ seam where both halves ship together and share the schema module. The fixture-pair pattern (candidate 2's shipped shape) covers the same risk in-repo. Prior art for the HTTP side remains [`docs/proposals/backlog/api-contract-enforcement.md`](../../../docs/proposals/backlog/api-contract-enforcement.md).
