The still-open concern: beta.97 cut `ELEVENLABS_MAX_ATTEMPTS` from 2 to 1 to reduce worst-case TTS latency. - 🏗️ `[LIFT]` **Validate `/wallet/list` response against Zod schema at api-gateway boundary** — Surfaced 2026-04-26 by claude-bot review on PR #908. The canonical `ListWalletKeysResponse` schema declares `timestamp: z.string()` (required) and per-key `createdAt` / `lastUsedAt`. The api-gateway handler at `services/api-gateway/src/routes/wallet/listKeys.ts:51-59` hand-constructs the response shape and currently emits all required fields — so no observed runtime bug. But the type contract is enforced by convention, not runtime check: a future refactor that drops `timestamp` (or any other required field) would silently break bot-client consumers using `ListWalletKeysResponse` without typecheck or test catching it. **Fix shape**: call `ListWalletKeysResponseSchema.parse(payload)` before `sendCustomSuccess` (or wrap `sendCustomSuccess` so route handlers pass the schema and parse happens centrally). Same gap applies to other gateway endpoints with hand-coded responses against schemas in `common-types/schemas/api/`. Folds naturally into the Observability & Telemetry theme's API-contract concerns. Surfaced 2026-04-26.
