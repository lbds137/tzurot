- 🐛 `[FIX]` **Add `Retry-After` header on `RedisRateLimiter` 429 responses** — Surfaced 2026-05-17 by PR #1046 review. `RedisRateLimiter.checkRateLimit` sends `retryAfter` in the JSON response body but doesn't set the standard `Retry-After` HTTP response header. RFC 6585 §4 says 429 responses SHOULD include `Retry-After` so HTTP clients (curl, axios with retry, browser fetch) can back off automatically without parsing JSON. `resetTime` is already computed in the handler, so the fix is a one-line addition: `res.set('Retry-After', String(resetTime))` before the `.status().json()` call. Affects all consumers of `RedisRateLimiter` (wallet, denylist, the new public-route limiter from PR #1046). **Start**: `services/api-gateway/src/utils/RedisRateLimiter.ts` `checkRateLimit` method, ~line 117.
