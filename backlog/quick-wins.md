- 🧹 `[CHORE]` **Drop redundant `provider: 'openrouter'` overrides in `createMockPersonality` test-fixture call sites** — Surfaced 2026-04-27 in PR #924 round 7 (claude-review). PR #924 added `provider: 'openrouter'` to the `createMockPersonality` factory base fixture, so the ~80 inline overrides like `createMockPersonality({ model: 'gpt-4o', provider: 'openrouter' })` are now redundant — the factory already supplies it. **No correctness impact, just noise.** **Fix shape**: `grep -rn "provider: 'openrouter'" services/ packages/` then for each match in a `createMockPersonality(...)` arg object, drop the `provider:` line if the factory default is what's wanted. Skip overrides that intentionally test against a different provider (rare). ~80 fixture sites, mechanical sweep, single PR. Tests should remain green throughout — it's pure noise reduction.

- 🧹 `[CHORE]` **Skip `claude-review` workflow on dependabot PRs** — Surfaced 2026-04-27 during dependabot triage of PRs #922/#923. The `claude-review` action structurally rejects bot actors with `Action failed: Workflow initiated by non-human actor: dependabot (type: Bot)`, so every dependabot PR shows a red `claude-review` check that's pure infra noise (claude-review on a `package.json` version-bump diff would be low-signal anyway). **Fix shape**: add `if: github.event.pull_request.user.login != 'dependabot[bot]'` (or equivalent) to the claude-review job in `.github/workflows/claude-review.yml`. Job-level `if:` cleanly skips for bot-authored PRs without producing a fail. **CRITICAL — release-coupled timing**: the claude-review action self-validates that the workflow file on the PR branch matches `main`; any divergence between develop and main on this file fails the action on _every_ subsequent PR until the develop→main release merge syncs them. **Schedule this as the LAST commit on develop before the next release-merge PR opens**, so the dark window where claude-review is broken on develop is minimized. Sized as ~5min code change + a release-checklist note. Until landed, dependabot PRs require explicit "merge anyway" per `00-critical.md`.

- 🧹 `[CHORE]` **Make `guard:duplicate-exports` and `knip` checks blocking in CI** — Surfaced 2026-04-27 during PR #918 work. Both checks currently have `continue-on-error: true` in `.github/workflows/ci.yml` so they print warnings but don't fail the lint job. Result: real findings get treated as informational and easily missed (e.g., PR #918's first push had `StartupDMPrewarmerDeps` flagged by knip and would have been merged uncaught if the contract-audit hadn't independently caught the schema gap). **Why deferred**: enforcement requires building an allowlist for legitimate duplicates first. As of 2026-04-27 the duplicate-exports check shows 10 groups — most are intentional (mocks under `__mocks__/` duplicating production exports per Vitest pattern, re-exports through `index.ts` barrels). Triaging them is a focused 30-60 min task. **Fix shape**: (a) audit `pnpm ops guard:duplicate-exports` output, classify each group as legitimate/incidental, (b) extend the script's allowlist (`packages/tooling/src/dev/check-duplicate-exports.ts` `ALLOWLIST` constant) to cover the legitimate ones, (c) audit `pnpm knip` output the same way, configure `knip.json` ignores for intentional internal exports, (d) flip `continue-on-error: true` → `false` (or remove the line) in both CI steps. Once enforced, reviewers + CI both gain a real signal on dead code and unintended duplicates.
