_Small tasks that can be done between major features. Good for momentum._

> Note: 7 items previously filed here all shipped in PR #1082-1084 (Layer 2 + Layer 3 of the periodic-audit-enforcement proposal). The remaining work tracked in [`docs/proposals/backlog/periodic-audit-enforcement.md`](../docs/proposals/backlog/periodic-audit-enforcement.md) is Layers 4-5 (markdown baselines + `ops:health` cron aggregator).

- **[LIFT] Cover `view.ts` `handleExpandField` + `handleViewPagination` error path** — PR #1132 added `handleView` / `handleViewPagination` happy + 404 coverage but left two gaps: (1) `handleExpandField` (view.ts ~428-470, wired in `dashboard.ts`, mocked at the import boundary in `dashboard.test.ts` so its body never executes) has 5 untested branches — found → `sendChunkedReply`, character-not-found, unknown-field, empty-content → `_Not set_`, catch → error. (2) `handleViewPagination`'s catch block (non-404/403 error → silent log, keep existing content) is untested. **Fix shape**: reuse the `gatewayClientStubs` + `characterViewOptions`/`clientsFor` mock pattern now in `view.test.ts`; `handleExpandField` needs `deferReply` alongside `editReply` on the mock interaction. ~40-60 LOC. Surfaced by PR #1132 claude-review.

- **[LIFT] Type the preset modal-unflatten pipeline to drop the `Parameters<>` casts** — `updatePreset`/`updateGlobalPreset` (`services/bot-client/src/commands/preset/api.ts`) take `data: Record<string, unknown>` and cast it to `Parameters<UserClient['updateUserLlmConfig']>[1]` at the client call. The cast exists because the upstream `unflattenPresetData` (`commands/preset/config.ts`) builds the update payload dynamically from flattened modal string values and returns `Record<string, unknown>`. **Fix shape**: type `unflattenPresetData`'s return as `LlmConfigUpdateInput` (already exported from common-types) by typing the `advancedParameters` sub-object construction, then retype the two `updateXxx` params to `LlmConfigUpdateInput` and remove the casts. Genuinely separate from the PR-F output-schema tightening (which removed the response-side `as PresetData` casts) — this is the *input* side and requires typing the dynamic modal pipeline. Surfaced 2026-06-02 during PR #1135 (the inbox LlmConfigSummary-tightening item conflated input + output cast removal).

- **[LIFT] Guard: service Dockerfiles must copy every runtime workspace dep's dist** — Each service `Dockerfile` runner stage copies workspace-package `dist` dirs **manually** (one `COPY --from=builder /app/packages/<pkg>/dist` per runtime `@tzurot/*` dependency). When `@tzurot/clients` was extracted (PR-2m) and bot-client gained it as a runtime dep, the COPY was never added → bot-client crash-looped in dev with `ERR_MODULE_NOT_FOUND` (fixed in PR #1145). Standard CI doesn't build the Docker image, so nothing caught it until the deploy. **This bug class recurs on every package extraction.** **Fix**: add a `pnpm ops guard:dockerfile-dist` check (CI lint job) that, for each `services/*/Dockerfile`, parses the prod `@tzurot/*` dependencies from the matching `package.json` and asserts each has a corresponding `COPY .../dist` line in the runner stage (and flags stale copies for removed deps). Static string/regex check, no Docker build needed — runs in the existing fast `lint` job. Mirrors the existing `guard:*` tooling pattern. Surfaced 2026-06-03 by the PR #1145 bot-client dev crash. **Sibling (not guardable in CI):** the Railway per-service *watch paths* are the same "per-service config that isn't auto-synced on package extraction" class, but dashboard-only — documented in [`docs/reference/deployment/RAILWAY_OPERATIONS.md`](../docs/reference/deployment/RAILWAY_OPERATIONS.md#watch-paths-per-service-redeploy-triggers) instead (broad `packages/**` today, so package extraction needs no change — but monorepo structure changes do).
