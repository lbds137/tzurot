## ⚡️ Quick Wins

_Small tasks that can be done between major features. Good for momentum._

- **[CHORE] Migrate `/admin/db-sync` + `/admin/cleanup` to `X-User-Id` header** — both routes pass `{ownerId}` in POST body, which is why `extractOwnerId` retains a body-fallback path. All other admin routes use the `X-User-Id` header convention (post-PR #1081). Migrating these two would let the body fallback come out of `extractOwnerId` entirely. **Start**: `services/bot-client/src/commands/admin/db-sync.ts:137-140` + `commands/admin/cleanup.ts:31` — drop the `ownerId` body field, rely on `adminPostJson` setting the header. Then drop the body block in `services/api-gateway/src/services/AuthMiddleware.ts:extractOwnerId`. ~10 LOC. Surfaced by PR #1081 review.

- **[CHORE] Improve `cpd:check` drift error message to mention threshold mismatch** — when `cpd:check --threshold 0.7` runs against a baseline captured at the default `0.8`, the configHash mismatches and the tool fails with "Run `cpd:update-baseline` to refresh." A user who deliberately wants to check at a non-default threshold (without updating the baseline) gets misleading guidance — running `cpd:update-baseline` without `--threshold` re-captures at the default. **Start**: in `packages/tooling/src/commands/cpd.ts:runCheckCommand`, when emitting the drift error, also print "If you're running `cpd:check` with a non-default `--threshold`, confirm it matches the baseline's `threshold` field." Surfaced by PR #1084 round-7 review.

- **[CHORE] Add `--summary` mode to `guard:boundaries`** — `guard:boundaries` is registered as audit-class in `AUDIT_TOOL_REGISTRY`, but the command doesn't yet support `--summary`. Inconsistent with the other registered guards (`guard:proposal-links`, `guard:audit-tool-docs`), both of which emit JSONL summary lines. The Layer-5 aggregator (when built) will skip this tool until it does. **Start**: in `packages/tooling/src/commands/guard.ts:guard:boundaries`, add the `--summary` option matching the existing pattern. In `packages/tooling/src/dev/check-boundaries.ts`, gate the existing terminal output behind `!options.summary` and emit a JSONL line via `emitSummary` when `--summary` is passed (status: 'fail' if violations exist, 'ok' otherwise; findings count = violations.length). Surfaced by PR #1085 review (the doc inconsistency between 05-tooling.md and the registry table led to discovering this gap).
