## ⚡️ Quick Wins

_Small tasks that can be done between major features. Good for momentum._

- **[CHORE] Add `--summary` mode to `guard:boundaries`** — `guard:boundaries` is registered as audit-class in `AUDIT_TOOL_REGISTRY`, but the command doesn't yet support `--summary`. Inconsistent with the other registered guards (`guard:proposal-links`, `guard:audit-tool-docs`), both of which emit JSONL summary lines. The Layer-5 aggregator (when built) will skip this tool until it does. **Start**: in `packages/tooling/src/commands/guard.ts:guard:boundaries`, add the `--summary` option matching the existing pattern. In `packages/tooling/src/dev/check-boundaries.ts`, gate the existing terminal output behind `!options.summary` and emit a JSONL line via `emitSummary` when `--summary` is passed (status: 'fail' if violations exist, 'ok' otherwise; findings count = violations.length). Surfaced by PR #1085 review (the doc inconsistency between 05-tooling.md and the registry table led to discovering this gap).

- **[CHORE] Codegen `mounts.ts` should merge same-module imports** — when multiple route IDs map to the same source file in `handler-paths.ts`, the codegen emits one `import { handleX } from '../foo.js'` line per handler instead of a single grouped statement. Example output observed in PR #1110 for `channel/configOverrides.js` (3 separate lines) and pre-existing for `user/channel/get.js` (2 separate lines). Functionally harmless (bundler/Node dedup), but cleaner output. **Start**: in `packages/tooling/src/codegen/mounts-builder.ts` `buildImports`, group entries by their resolved source path before emitting. Each group becomes `import { X, Y, Z } from '../path.js'`. Surfaced repeatedly across PR #1110 review rounds. Surfaced 2026-05-28.

- **[LIFT] Consolidate `extractOwnerId` + `extractUserId` in AuthMiddleware** — both read the `x-user-id` header and now differ only in the empty-string guard: `extractUserId` has `&& headerUserId.length > 0` (returns `undefined` for an empty header) while `extractOwnerId` returns `''`. Net auth behaviour is identical — `isValidOwner` checks `length === 0` and `requireUserAuth` checks `length === 0`, so both fail closed on an empty header — making this a readability hazard, not a bug. **Start**: `services/api-gateway/src/services/AuthMiddleware.ts` — merge into one header-extraction helper (or have `extractOwnerId` delegate to `extractUserId`) so the empty-header handling is single-sourced; update the two call sites. ~10 LOC. Surfaced 2026-06-01 by PR #1126 claude-review (flagged as pre-existing, non-blocking).
