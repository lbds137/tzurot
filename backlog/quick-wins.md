## ⚡️ Quick Wins

- 🧹 `[CHORE]` **Make `guard:duplicate-exports` and `knip` checks blocking in CI** — Surfaced 2026-04-27 during PR #918 work. Both checks currently have `continue-on-error: true` in `.github/workflows/ci.yml` so they print warnings but don't fail the lint job. Result: real findings get treated as informational and easily missed (e.g., PR #918's first push had `StartupDMPrewarmerDeps` flagged by knip and would have been merged uncaught if the contract-audit hadn't independently caught the schema gap). **Why deferred**: enforcement requires building an allowlist for legitimate duplicates first. As of 2026-04-27 the duplicate-exports check shows 10 groups — most are intentional (mocks under `__mocks__/` duplicating production exports per Vitest pattern, re-exports through `index.ts` barrels). Triaging them is a focused 30-60 min task. **Fix shape**: (a) audit `pnpm ops guard:duplicate-exports` output, classify each group as legitimate/incidental, (b) extend the script's allowlist (`packages/tooling/src/dev/check-duplicate-exports.ts` `ALLOWLIST` constant) to cover the legitimate ones, (c) audit `pnpm knip` output the same way, configure `knip.json` ignores for intentional internal exports, (d) flip `continue-on-error: true` → `false` (or remove the line) in both CI steps. Once enforced, reviewers + CI both gain a real signal on dead code and unintended duplicates.
