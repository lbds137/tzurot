# Research Notes Archive

> **Purpose**: Distilled insights from AI consultations, technology research, and external documentation analysis.

## What Goes Here

- **TL;DR summaries** of AI consultation sessions (Gemini, Claude, etc.)
- **Technology research** for potential integrations
- **Competitive analysis** of similar projects

## File Format

Each research note follows this structure:

```markdown
# [Topic]

> **Date**: YYYY-MM-DD
> **Source**: [Original consultation date/context]
> **Status**: Active/Archived

## TL;DR (10 lines max)

[Quick summary for AI context loading]

## Key Findings

[Detailed insights organized by topic]

## Actionable Items

[Link to BACKLOG.md items where applicable]
```

## Why Not Keep Raw Transcripts?

1. **Context bloat** - Raw transcripts are 20-200KB vs 2-5KB distilled notes
2. **Searchability** - Structured notes are easier to search and reference
3. **Git history** - Original transcripts are preserved in git history if needed
4. **Maintenance** - Fewer files = easier to keep current
