- **PR #952** (`f17764876`) — `fix(ai-worker): normalize CRLF in TTS chunker to match FormData wire size`. Root-cause fix for production voice synthesis 400s ("Text too long, 2001-2006 chars" against the 2000-char cap, observed 2026-04-27 and 2026-04-30). Council analysis (Gemini 3.1 Pro Preview) traced this to **WHATWG `multipart/form-data` spec (RFC 7578 §4.4) mandating `\n` → `\r\n` normalization on the wire**. Node's built-in `FormData` implements this, so a JS-side 2000-char chunk with N bare `\n` chars expands to 2000+N chars by the time voice-engine measures it. Both historical sightings fit the same model: `off-by-N == newline count`. Two-sided fix: TS chunker normalizes `\n` → `\r\n` at the entry of `splitTextIntoChunks` (using `/\r?\n/g` to prevent `\r\r\n` drift on already-CRLF input); voice-engine measures visible chars (text minus `\r`) so spec-compliant CRLF callers aren't penalized. Logging changed from misleading `textLength: text.length` to `wireLength: chunks[0].length` (single-chunk) / `maxChunkLength: Math.max(...)` (multi-chunk) for direct diagnosability of future 400s. **8 review rounds** — rounds 1-3 produced substantive improvements (test hardening, inline `\r` strip eliminating a deferred backlog entry, tighter regex assertions, boundary-value test correction); rounds 4-5 added genuine polish (log-field rename, paired icebox edge-case framing, idiomatic vitest matchers, identity-contract + sentence-boundary tests); rounds 6-8 were reviewer-mode-decay (repeating already-dismissed nits, contradicting round-1 corrections, surfacing pure style observations). 8 fixup commits autosquashed at pre-merge rebase. Backlog hygiene: paired CRLF + surrogate-pair edge cases under one icebox entry; scoped out the previously-paired UTF-8-byte hypothesis (disproved by this PR's actual root cause).
